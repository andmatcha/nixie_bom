# KiCad 電源回路図配線メモ

作業日: 2026-06-23

対象ファイル:

- `kicad/nixie_clock/nixie_clock.kicad_sch`
- `tools/generate_kicad_bom_project.py`
- `tools/wire_kicad_power_schematic.py`

## 再適用方法

電源系の部品配置、直配線、必要最小限のネットラベルは、次のコマンドで再生成できる。

```sh
python3 tools/wire_kicad_power_schematic.py
```

このスクリプトは、以前に自分が生成したワイヤ、ラベル、ジャンクションを UUID で削除してから再生成する。KiCad で配置を調整したあとに再適用する場合は、同じ電源部品のシンボル参照名を維持する。

## 配線方針

機能ブロックごとの配置は維持しつつ、電源系は部品間の関係を目で追えるように、電源レール、GND、帰還、補償、スイッチングノード、電流検出、3.3 V Buck 周辺をできるだけワイヤで直接接続した。

ネットラベルは、外部入力、ブロック外へ出ていく電源、または長距離にまたがる代表ネットを示す用途に限定した。ラベルだけで接続を成立させるのではなく、電源ブロック内のローカルな部品同士はワイヤでつなぐ。

KiCad は、見た目でワイヤが交差しているだけでは接続と見なさない一方、同一直線上にワイヤを重ねると意図しない同一ネットや `label_multiple_wires` 警告につながる。分岐点にはジャンクションを置き、重なりワイヤを作らないようにした。

今回の対象は電源生成部であり、表示管、シフトレジスタ、ドライバ、CAN、MCU などの下流回路への給電は `+3V3`、`HV170`、`GND` などのネットラベルとして引き出すところまでにした。`HV_DISABLE` は将来 MCU またはテストポイントへ接続する外部制御ネットとして残している。

## 主なネット

| ネット名 | 意味 |
| --- | --- |
| `+12V_IN` | 外部 12 V 入力。3.3 V Buck と HV 分岐ヒューズの入力。 |
| `+12V_HV` | `FHV1` 後段の HV Boost 用 12 V レール。 |
| `HV170` | ニキシー管用の約 170 V 出力。 |
| `SW_HV` | TPS40210 昇圧段のスイッチングノード。 |
| `ISENSE_SRC` | MOSFET ソース側の電流検出ノード。 |
| `ISNS` | TPS40210 の ISNS 入力。`RIFLT1`/`CIFLT1` 後段。 |
| `DIS_EN` | TPS40210 の DIS/EN ピン周辺ノード。 |
| `HV_DISABLE` | 外部制御入力。High で TPS40210 を停止させる前提のため、この名前にした。 |
| `+3V3` | AP63203QWU-7 の固定 3.3 V 出力。 |
| `SW_3V3` | 3.3 V Buck のスイッチングノード。 |
| `BST_3V3` | 3.3 V Buck のブートストラップノード。 |
| `BUCK_EN` | AP63203QWU-7 の EN ノード。 |

## HV Boost の接続

- `FHV1` は `+12V_IN` と `+12V_HV` の間に入れ、HV Boost 分岐だけを保護する。
- `L1` は `+12V_HV` と `SW_HV` の間に接続する。
- `Q1` はゲートを `RG1` 経由で `GDRV`、ドレインを `SW_HV`、ソースを `ISENSE_SRC` に接続する。4 ピン NMOS シンボルのドレイン 2 本は両方 `SW_HV` にした。
- `D1` は KiCad 標準 `Device:D` の pin 1 が K、pin 2 が A なので、pin 1 を `HV170`、pin 2 を `SW_HV` に接続した。
- `CIN1`、`CIN2`、`CINB1`、`CINB2`、`CVDD1` は `+12V_HV` と `GND` 間に置く。
- `COUT1`、`COUT2`、`COUT3` は `HV170` と `GND` 間に置く。
- `RFB1` から `RFB5` は 20 kOhm x5 の直列上側分圧、`RFB6` は `FB` から `GND` への下側分圧とした。
- `RBLD1` から `RBLD3` は `HV170` から `GND` への直列ブリーダ抵抗とした。
- `RCOMP1` と `CCOMP1` は `COMP` と `FB` の間に直列、`CHF1` は `COMP` と `FB` の間に並列で入れた。
- `RRC1` は `+12V_HV` と `RC`、`CRC1` は `RC` と `GND` 間に入れた。
- `CSS1` は `SS` と `GND`、`CBP1` は `BP` と `GND`、`CIFLT1` は `ISNS` と `GND` 間に入れた。
- `RENPD1` は `DIS_EN` を `GND` に落とすので、外部制御がないときは HV Boost が有効になる。
- `RENPD1` は制御系ではなく TPS40210 の DIS/EN 周辺部品として扱うため、KiCad 生成元のブロック分類も HV Boost 側へ変更した。

## 3.3 V Buck の接続

- `U3` は KiCad 標準の `Regulator_Switching:AP63200WU` を AP63203QWU-7 の代替シンボルとして使う。
- AP63203QWU-7 は固定 3.3 V 出力なので、`FB` は `+3V3` に接続する。
- `IN` は `+12V_IN`、`GND` は `GND`、`SW` は `SW_3V3`、`BST` は `BST_3V3` に接続する。
- `L2` は `SW_3V3` と `+3V3` の間に接続する。
- `C3V3IN1`、`C3V3IN2`、`C3V3HF1` は `+12V_IN` と `GND` 間に置く。
- `C3V3OUT1`、`C3V3OUT2` は `+3V3` と `GND` 間に置く。
- `CBOOT1` は `BST_3V3` と `SW_3V3` 間に置く。
- `R3V3EN1` は `+12V_IN` から `BUCK_EN` へのプルアップとし、3.3 V 電源を常時有効にする。

## 作業で得られた知見

- TPS40210 の DIS/EN は High で停止、Low または未接続で有効になる。既存文書の `HV_ENABLE` という表現は機能名としては自然だが、回路図ネット名は論理を取り違えないよう `HV_DISABLE` とした。
- TPS40210 の `VDD` は 170 V 側ではなく 12 V 側へ接続する。今回は HV 分岐ヒューズ後段の `+12V_HV` にした。
- TPS40210 の `BP` は内部レギュレータ出力なので、外部負荷へ出さず `CBP1` で GND にバイパスする。
- `RFB1` から `RFB5` と `RBLD1` から `RBLD3` は、同じ値の抵抗を複数並べているが並列ではなく直列接続である。中間ノード名を分けて直列であることを明示した。
- AP63203QWU-7 は固定出力品のため、FB を外付け分圧にせず `+3V3` へ直接戻す。
- KiCad のシンボルライブラリ内のローカル座標は Y 軸が上向きだが、回路図シート座標は下向きに増える。スクリプトでピン位置を計算するときは `global_y = symbol_y - pin_y` として扱う必要がある。
- KiCad では、ピンがワイヤの途中に見た目上重なっていても、ワイヤ端点またはジャンクションがないと接続されない場合がある。電源レールへの枝分かれ点には明示的にジャンクションを置く。
- ヒューズ前後の `+12V_IN` と `+12V_HV` は、ヒューズのシンボルを挟むだけでなく、ワイヤが重なっていないことを netlist で確認する必要がある。後段レールをヒューズ入力側まで横切らせると、見た目では自然でも前後が短絡する。
- ラベルは実ワイヤ上に置く必要がある。ワイヤから少しでも外れると `label_dangling` になり、重なりワイヤ上に置くと `label_multiple_wires` になる。
- `kicad-cli` は PATH にはないが、`/Applications/KiCad/KiCad.app/Contents/MacOS/kicad-cli` から実行できる。SVG エクスポートで電源ブロックの描画を確認し、netlist で `+12V_IN` と `+12V_HV` が分離していること、電源生成部の主要部品が未接続ネットに残っていないことを確認した。
- ERC には、未配線の表示管、シフトレジスタ、ドライバ、MCU、CAN 由来の警告が残る。電源部では `HV_DISABLE` が外部制御予定の片側ラベルのため `isolated_pin_label` として残るが、これは MCU 側またはテストポイントを配線する段階で解消する。
- KiCad を開いたままファイルを変更している場合は、KiCad 側で再読み込み確認が出ることがある。保存競合を避けるため、再適用前に KiCad 側の未保存変更を保存しておく。
